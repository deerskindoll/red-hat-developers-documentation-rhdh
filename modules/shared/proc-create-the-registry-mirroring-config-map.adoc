= Create the registry mirroring config map

[role="_abstract"]
Create a config map containing the mirroring configuration to enable {product-short} to pull plugins from your mirror registry instead of the original source registries.

.Procedure
* Create a config map containing the registry mirroring configuration:
+
[source,yaml]
----
apiVersion: v1
kind: ConfigMap
metadata:
  name: my-rhdh-mirror-conf
data:
  rhdh-registries.conf: |
    [[registry]]
    # GA and TP plugins
    prefix = "registry.access.redhat.com/rhdh"
    location = "my-registry.example.com/rhdh"

    [[registry]]
    prefix = "quay.io/rhdh"
    location = "my-registry.example.com/rhdh"

    [[registry]]
    # Community plugins
    prefix = "ghcr.io/redhat-developer/rhdh-plugin-export-overlays"
    location = "my-registry.example.com/rhdh-plugin-export-overlays"
----
+
[NOTE]
====
While GA and Technical Preview plugins can be pulled from `registry.access.redhat.com`, community-supported plugins might be available from other registries such as `ghcr.io` or `quay.io`. You can configure mirroring for those registries as well. Make sure to adjust the `location` paths accordingly, depending on your target mirror registry.
====
