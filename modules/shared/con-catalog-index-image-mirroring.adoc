= Catalog index image mirroring

[role="_abstract"]
When you use the `--plugin-index` option, the mirroring script mirrors the plugin catalog index image itself in addition to the individual plugin artifacts. The mirrored location is included in the generated `rhdh-plugin-mirroring-summary.txt` file.

If you configure transparent registry mirroring using `registries.conf`, as described in the creating registry mirroring ConfigMap procedure, no additional {product-short} configuration is needed. The catalog index image is pulled from the mirror automatically.

Alternatively, you can explicitly point {product-short} to the mirrored catalog index images by setting `CATALOG_INDEX_IMAGE` or `EXTRA_CATALOG_INDEX_IMAGES` environment variables on the `install-dynamic-plugins` init container. You can configure these environment variables using `extraEnvs` in the {backstage} custom resource when using the {product-short} Operator, or using `upstream.backstage.initContainers.env` in Helm values when using the {product-short} Helm chart. However, this configuration is not necessary when `registries.conf` is properly configured for transparent mirroring.

[NOTE]
====
Unlike container images pulled by the Kubelet, plugin catalog index images are pulled directly by `skopeo` from inside the `install-dynamic-plugins` init container. This means that cluster-level mirroring resources such as `ImageContentSourcePolicy`, `ImageDigestMirrorSet`, and `ImageTagMirrorSet` have no effect. You must use the `registries.conf`-based approach.
====
